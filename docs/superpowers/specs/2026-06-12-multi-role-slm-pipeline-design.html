Multi-Role SLM Pipeline

SpecGuard design · 2026-06-12 · first slice of the SLM-first agentic harness. The harness — not the model — is the product; each SLM is a replaceable worker inside it.

Scope

In this slice
  • Planner / Writer / Critic role modules
  • Deterministic Router with retry budget (default 3)
  • Scored rubric JSON from the Critic, tolerant parsing
  • Per-role model config with shared default (gemma4 / Ollama)
  • New pipeline events: plan, attempt, critique
  • Draft-with-warnings degradation instead of hard failure
Later slices
  • Memory/state store across sessions
  • Policy / approval layer
  • Eval harness (fixed prompts, regression scoring)
  • Retiring agent.py
  • Ask-user interrupts when the loop is stuck

Roles

SLM ×1 + tools

Planner

Runs clarifier, domain_checklist, scope_estimator as plain Python (no tool-calling protocol), then one SLM call to synthesize a writing brief: assumptions, inferred answers, per-section guidance.

SLM per attempt

Writer

Drafts from idea + brief + mode standard. Same role handles revisions — the revision prompt carries critic findings and missing-section list.

SLM per attempt

Critic

Returns scored rubric JSON: per-criterion {id, score 0–2, reason}, overall verdict, free-text notes for the Writer. Rubrics restructured from rubrics.py.

Pure Python — no SLM

Router

decide(validation, verdict, attempt, budget) → finalize / revise / degrade. Combines deterministic validators with the critic verdict. Never silently blocks output.

Critic JSON robustness

  1. Tolerant extraction — strip code fences, locate the first {...} object, json.loads.
  2. Retry once on parse failure.
  3. Fallback — validator-only gating for that iteration, recorded as critique event with fallback: true.

The loop never dies because the critic emitted bad JSON.

Module layout

  pipeline.py        # orchestrator; public contract unchanged
  router.py          # pure decision function
  roles/
    planner.py       # plan(model, idea, mode, standard) -> Brief
    writer.py        # draft(...) -> str, revise(...) -> str
    critic.py        # critique(model, mode, draft) -> CriticVerdict

  config.py          # + per-role model fields
